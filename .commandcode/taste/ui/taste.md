The chatbot should know and answer directly about the core tech stack (React.js, TypeScript, Tailwind CSS, Node.js & Express.js, PostgreSQL & Supabase, Docker & Vercel Deployment). Confidence: 0.85

- Chatbot responses must follow a strict Markdown formatting structure: (1) every heading (`###`) isolated on its own line with a blank line before and after — never merged into previous text; (2) bullet lists use `- ` prefix with each item on its own line, blank line before/after the list, and items under 15 words; (3) bold labels at the start of key lines (`**Skill:** React.js`) for scannability; (4) sentences kept under 15 words with no wall-of-text paragraphs; (5) blank lines between every section. On the frontend, the ChatMarkdown component must explicitly render headings (`h1`-`h4`) with proper margin/spacing classes so the LLM's Markdown spacing isn't collapsed into a single text block. Confidence: 0.85
- When a scrollable container (`overflow-y-auto`) is inside a `flex-col` parent, always add `min-h-0` (or equivalent `min-height: 0`) to the scrollable child. This overrides flexbox's default `min-height: auto` which otherwise prevents the child from shrinking below its content height, killing the scroll. Confidence: 0.85
- On scrollable containers inside overlays/panels (chatbot messages, modals, etc.), add `overscroll-contain` to prevent touch scroll chaining where swiping at the container's scroll boundary scrolls the page behind it. Confidence: 0.80
- For scrollable areas inside dark-themed overlays (chatbot, modals, sidebars), style scrollbars with a thin profile (4px width, `scrollbar-width: thin`), transparent track, and a faint semi-transparent white thumb that becomes more visible on hover — instead of completely hiding scrollbars. This provides a professional feel while keeping the UI clean. Confidence: 0.80

- When a sidebar or panel contains a scrollable messages/content area, forward mouse wheel (`onWheel`) and touch (`onTouchStart`/`onTouchMove`) scroll events from the entire sidebar surface to the internal scrollable container, so scrolling works regardless of which part of the sidebar the user interacts with. Check scroll boundaries so events pass through naturally when the container is at the top or bottom. Confidence: 0.85
