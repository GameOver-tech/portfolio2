# workflow
- When fixing a production-only bug (works locally, fails after deployment), diagnose it as a deployment configuration issue first — not a code logic issue. The methodology: if the same credentials/code path works on localhost but fails in production, the root cause is almost certainly in environment variables, CORS settings, API URL config, platform env var injection, or Supabase RLS/network policies — not in the authentication logic itself. Confidence: 0.90
- When asked to fix a bug, apply **minimal, surgical fixes only**: do NOT refactor, redesign, restructure folders, rename variables, or modify any code unrelated to the bug. Never introduce breaking changes, never modify unrelated files, never remove existing authentication/functionality, never change database schema unless absolutely required. Keep the project stable. Fix only what is necessary. Confidence: 0.90
- When fixing a mobile responsiveness or layout issue on one page, proactively audit and fix ALL other pages that have the same class of issue in a single pass — do not fix pages individually one at a time. The user expects comprehensive, cross-page fixes rather than repeated per-page requests. Confidence: 0.8
- When debugging a production deployment failure (especially authentication/login issues), perform a **comprehensive, multi-layer audit** following a systematic methodology: (1) check frontend API requests (base URL, Axios config, hardcoded URLs, trailing/double slashes, HTTPS); (2) inspect frontend env vars; (3) inspect backend env vars; (4) inspect CORS configuration; (5) inspect JWT/auth flow (creation, verification, expiration, storage, headers); (6) inspect Supabase connection, keys, RLS policies, tables; (7) inspect the login API route (payload, response, status codes, error handling, bcrypt compare); (8) inspect frontend login flow (form, Axios request, token storage, redirect); (9) check browser console + network tab; (10) check backend logs; (11) verify production build has no localhost references; (12) inspect deployment platform env vars and logs; (13) inspect cookie config (sameSite, secure, httpOnly); (14) inspect auth middleware; (15) trace the full login chain end-to-end to find exactly where it breaks. Do NOT guess or jump to conclusions — inspect every file methodically before proposing a fix. Confidence: 0.85
- When reporting findings for a bug fix, deliver a structured report with exactly: (1) Problem Found, (2) Root Cause (including why localhost works vs. why production fails), (3) Files Affected (with exact file name and line number), (4) Exact Code Changes (Before code block + After code block), (5) Why This Fix Works, (6) Deployment Checklist, (7) Testing Steps, (8) Confirmation that localhost behavior remains unchanged. Do NOT skip any of these sections. Confidence: 0.85
- When debugging layout/styling bugs, perform a **systematic, exhaustive audit** following a structured methodology: (1) scan every component file; (2) inspect every CSS file for problematic properties (position, z-index, overflow, transform, clip-path, negative margins, backdrop-filter, height: 100vh); (3) inspect every parent container for document flow correctness; (4) check all responsive breakpoints (320px, 375px, 425px, 768px, 1024px, 1440px, 1920px); (5) check animations for layout-shifting side effects; (6) inspect the full z-index hierarchy across the app; (7) explain why each dangerous CSS usage is safe or unsafe; (8) fix every issue. Do not stop until every overlap and blur issue is eliminated. Confidence: 0.85
- When debugging blurry text or mobile rendering issues, search systematically for ALL these CSS properties on text-containing elements: `filter: blur()`, `backdrop-filter`, `transform: translateZ()`, `transform: scale()`, `opacity` (sub-1), `mix-blend-mode`, `will-change`, `perspective`, `overflow: hidden`, and `z-index`. Enforce a strict CSS separation: every heading and paragraph must have `filter: none; backdrop-filter: none; transform: none; opacity: 1; position: relative; z-index: 2;` while every decorative glow/blob element must have `position: absolute; z-index: 0; pointer-events: none;`. Content containers should use `position: relative; z-index: 10;` to ensure they render above background effects. Confidence: 0.85
- When debugging layout issues, **preserve the existing visual design exactly** — do not change colors, fonts, spacing, animations, cards, or buttons. Only repair layout problems (overlaps, z-index conflicts, overflow, blur from covering layers, inconsistent spacing, positioning bugs). Confidence: 0.85
